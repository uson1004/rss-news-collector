# 읽을게 2차 보고서

## 1. 개발 진행 개요

1차 보고서에서는 접근성 좋은 리더뷰 MVP를 목표로, URL 입력 기반 본문 추출과 읽기 좋은 화면 구성을 핵심 기능으로 설정하였다.

2차 개발 단계에서는 단순한 본문 추출 기능을 넘어, 사용자가 실제로 읽을 콘텐츠를 더 쉽게 찾고 활용할 수 있도록 **뉴스 읽기, RSS 카테고리, AI 요약, TTS, 번역, 뉴스레터 기능**을 추가하였다.

현재 프로젝트는 **React 프론트엔드 + FastAPI 백엔드** 구조로 구현되었으며, URL 본문 추출과 RSS 기반 아티클 제공이 모두 하나의 리더뷰 화면으로 연결되도록 구성하였다.

## 2. 주요 구현 기능

**리더뷰 기능**

| 기능 | 구현 내용 |
| --- | --- |
| URL 본문 추출 | 사용자가 URL을 입력하면 서버에서 본문을 추출 |
| 단락 분리 | 본문을 문단 단위로 나누어 가독성 개선 |
| 리더뷰 화면 | 제목, 출처, 요약 설명, 본문을 읽기 좋은 형태로 표시 |
| 글자 크기 조절 | 사용자가 본문 글자 크기를 조절 가능 |
| 줄 간격 조절 | 줄 간격을 조절해 읽기 편의성 개선 |
| 문단 간격 조절 | 문단 사이 여백 조절 가능 |
| 본문 폭 조절 | 사용자 화면 환경에 맞게 본문 너비 조절 가능 |
| 다크모드 / 고대비 모드 | 밝은 화면, 어두운 화면, 고대비 화면 지원 |

**추가 기능**

| 기능 | 구현 내용 |

## 4. 화면 구성 변경

1차에서는 URL 입력 후 리더뷰로 이동하는 단순 흐름을 중심으로 구성하였다.

2차에서는 기능이 늘어나면서 화면 흐름을 다음과 같이 분리하였다.

```text
[상단 헤더]
 ├── 뉴스 읽기
 │    ├── RSS 카테고리 선택
 │    ├── 아티클 목록
 │    ├── 읽기 버튼
 │    └── 리더뷰 화면으로 이동
 │
 ├── URL로 본문 추출
 │    ├── URL 입력
 │    ├── 읽기 버튼
 │    └── 리더뷰 화면으로 이동
 │
 └── 공통 리더뷰 화면
      ├── 본문 표시
      ├── AI 요약
      ├── 질문 프롬프트 생성
      ├── 번역
      ├── TTS
      └── 읽기 설정 조절
```

기존에는 기능이 한 화면에 섞일 가능성이 있었지만, 현재는 **뉴스 읽기**와 **URL 본문 추출**을 분리하여 사용자가 목적에 맞게 접근할 수 있도록 개선하였다.

## 5. 개발 중 해결한 문제

| 문제 | 해결 방법 |
| --- | --- |
| 일부 웹사이트 본문 추출 실패 | 직접 HTML 추출이 실패하면 RSS/feed 기반 fallback 사용 |
| RSS 글의 링크가 실제 글 URL이 아닌 경우 | Atom/RSS의 `rel="alternate"` 링크를 우선 사용하도록 수정 |
| Google 뉴스 RSS는 리더뷰 추출에 부적합 | 국내 RSS와 기술블로그 RSS 중심으로 변경 |
| 카테고리 글 품질 문제 | 카테고리별 키워드 필터와 최신순 정렬 적용 |
| 사용자가 RSS URL을 직접 찾기 어려움 | 카테고리 이름 + 검색 힌트 기반 RSS 후보 탐색 방식 도입 |
| 뉴스레터 저장 구조 필요 | SQLite 기반 카테고리/구독 저장소 구현 |
| 메일 발송 환경 차이 | SMTP 설정이 없으면 구독은 저장하고 안내 메시지 표시 |

---

## 6. 현재 제약 사항

- 외부 웹사이트가 봇 접근을 차단하면 본문 추출이 실패할 수 있음
- AI 요약 기능은 Claude API 키가 필요함
- 번역 기능은 DeepL API 키가 필요함
- 뉴스레터 실제 발송은 SMTP 설정이 필요함
- 현재 뉴스레터는 구독/테스트 발송 중심이며, 자동 정기 발송 스케줄러는 아직 구현되지 않음
- SQLite를 사용하고 있어 실제 서비스 확장 시 Supabase 같은 외부 DB로 전환이 필요함
- RSS 기반 콘텐츠는 각 사이트의 피드 제공 여부와 품질에 영향을 받음

## 9. 결론

2차 개발에서는 1차 MVP의 핵심이었던 **접근성 좋은 리더뷰**를 유지하면서, 사용자가 읽을 글을 더 쉽게 찾고 활용할 수 있도록 기능을 확장하였다.

특히 RSS 카테고리 브라우저, AI 요약, TTS, 번역, 뉴스레터 기능을 추가하면서 단순한 본문 추출 도구에서 **읽기 중심 콘텐츠 서비스**에 가까운 형태로 발전하였다.

아직 자동 뉴스레터 발송, 사용자 계정, 외부 DB 저장소 같은 고도화 작업은 남아 있지만, 현재 단계에서는 프로젝트의 핵심 목표인 **웹 콘텐츠를 더 읽기 쉽고 접근성 좋게 제공하는 MVP**를 구현한 상태이다.
